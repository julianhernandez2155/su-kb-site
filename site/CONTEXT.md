# site/ context

The Quartz v4 project that builds the public-facing GitHub Pages site. Source markdown lives in `content/`; built static site emits to `public/` (gitignored).

## Audience

A frontend developer or Quartz-aware agent modifying styling, layouts, or Quartz plugins.

## Workspace structure (once populated)

- `content/` — source markdown (what the export-tool writes into)
- `content/index.md` — site root meta-index (lists departments)
- `content/llms.txt` — root LLM crawler index (may be regenerated by an emitter plugin)
- `content/data-ai/` — first department's content tree
- `quartz/styles/` — clementine-derived CSS (SU color tokens, Sherman Sans `@font-face`)
- `quartz/static/` — fonts and images (Sherman Sans `.woff2`, syracuse-logo-orange.svg, etc.)
- `quartz/plugins/` — custom Quartz plugins (`emit-md-mirror.ts`, `emit-llms-txt.ts`)
- `quartz.config.ts` — site config + plugin manifest
- `quartz.layout.ts` — page layouts using clementine components

## Patterns used

- **Quartz v4** (pinned — not v5). Native `[[wikilink]]` parsing via `quartz/plugins/transformers/ofm.ts`
- **Four-element URL pattern**: every page reachable at both `/path/` (HTML, for humans) AND `/path.md` (raw markdown, for Claude `WebFetch`); root `/llms.txt` for skill orientation; per-page "Copy as markdown" button (v2)
- **Clementine styling**: SU navy/orange tokens (`--su-blue: #000E54`, `--su-orange: #F76900`) + Sherman Sans + section ordering from clementine.syr.edu

## When working here

- `npm run build` produces `public/`; `npm run serve` for local preview
- **Don't hand-edit `content/data-ai/*.md` directly** — those files are written by the export-tool. Confluence is the source of truth; edit there and re-export
- Custom Quartz plugins go in `quartz/plugins/`; wire them in `quartz.config.ts`

## Anti-patterns

- Don't upgrade to Quartz v5 yet (plugin ecosystem too immature as of 2026-05; see ADR-0001)
- Don't add schema.org / JSON-LD markup as a primary play (empirical evidence is weak — see the 2026-05-28 deep-research report)
- Don't fight Quartz's component model with custom JSX hacks. If clementine styling doesn't fit cleanly within 2 days, document an ADR and consider MkDocs-Material fallback (per Stage 2 decision gate in the plan)

**Lessons worth keeping**:
- Quartz v4 expects `fontOrigin: "local"` to mean "user provides @font-face in CSS" — the actual font name in config maps directly to CSS `font-family`. The pattern works as long as the @font-face declaration matches.
- PowerShell single-quoted here-strings `@'...'@` work for multi-line strings BUT the resulting string passed to native exes (like `git`) can still break on inner double quotes. Use Bash + heredoc for `git commit -m` with rich messages.
- `gh` OAuth tokens often lack `workflow` scope by default. The MCP GitHub plugin bypasses this because it has its own auth path. Future repo workflows: either run `gh auth refresh -s workflow` once, or use the MCP plugin for workflow-file pushes.
- GH Pages on the free plan requires public repos. Anticipate this; the plan already called for flipping to public before serving.

## Update — same day, third session: Quartz → thin renderer pivot

After the morning scaffold + afternoon Stage-1+2 ship, the evening session re-examined the architecture.

**Empirical wikilink-through-WebFetch test ran** (originally scheduled as Stage 8 of the plan). 3 representative corpus pages pushed to `_test-wikilinks/` in two versions (`with-wikilinks/` keeps `[[page-id - title|alias]]`; `with-relative/` rewrites to `[alias](./page-id.md)`). All 6 fetched via `raw.githubusercontent.com`. **Result: wikilinks survive WebFetch.** Small-model summarizer preserved `[[...]]` syntax verbatim, classified them correctly as "Obsidian wikilink", parsed page-id + title + alias, proposed follow-up URLs. Both formats produced equivalent LLM output. Test fixtures committed at `99e6d70`.

**The deeper question** — do wikilinks add VALUE for our consumers? Walked through it. Wikilinks valuable when target was Obsidian (graph view, backlinks, hand-editing). For our setup (Confluence authoritative, no human edits markdown, consumers are HTML + Claude WebFetch), wikilinks are vestigial. Image embeds `![[...|size]]` same story. Page-ID-prefixed filenames, GFM callouts `> [!note]`, YAML frontmatter all KEEP (not Obsidian-specific).

**AI council debate** — spawned 3 Agent subagents (Defender for Quartz, Advocate for custom build, Skeptic refusing to pick sides). R1 produced 3 opinionated positions + ~15 questions. R2 (after the empirical test) converged: Advocate's narrowed Option C and Skeptic's refined Option D collapsed to functionally the same answer — thin Python renderer + Jinja2 templates of `_design/`. Defender walked back the corpus-evidence argument; held only on bus-factor / handoff grounds.

**Decisions made**:
- [ADR-0002](decisions/0002-pivot-from-quartz-to-thin-renderer.md) supersedes [ADR-0001](decisions/0001-quartz-v4-as-ssg.md) — drop Quartz; thin Python renderer (~150 lines target, hard ceiling 300, escape hatch to MkDocs-Material)
- **8-field frontmatter schema** (down from 26): `title`, `description`, `page_id`, `department`, `source_url`, `last_modified`, `tags`, `audience`. Drop access-classification fields, contributors (PII), build diagnostics, vestigial classifier fields
- **Drop Obsidian-isms at the source** — export tool emits `[alias](./slug.md)` and `![alt](path)`, not `[[wikilinks]]` and `![[embeds]]`
- **Slug-only filenames** (page-id moves to frontmatter only)
- **Hard rule for salvage stage**: do NOT modify any file under `SU_AI_Intern/prototypes/su-kb-pipeline/`. Copy modules into `su-kb-site/export-tool/`, edit copies only.
